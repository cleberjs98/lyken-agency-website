# Lyken Agency Website — UX Flow & Navigation

## Purpose

This document defines the visitor experience, navigation behavior and conversion flow for the Lyken Agency website.

The site must not only look premium. It must guide the visitor through a strategic story.

---

## 1. Core Visitor Journey

The desired journey is:

```txt
Curiosity → Clarity → Trust → Desire → Action
```

The user should feel:

1. This looks different and premium.
2. I understand what Lyken does.
3. Lyken has strategy and method.
4. I want this level of quality for my brand.
5. I know how to contact Lyken.

---

## 2. Narrative Flow

The website experience follows:

```txt
Loading → Hero → Journey → Services → Process → Solutions → Why Lyken → Contact → Footer
```

---

## 3. Section Order

Recommended order:

```txt
1. Loading Experience
2. Navbar
3. Hero
4. Journey / Transformation Story
5. Services
6. Transformation Layers
7. Process
8. Solutions by Business Stage
9. Why Lyken
10. Final CTA / Contact
11. Footer
```

---

## 4. Loading Experience Role

The loading experience introduces the concept:

```txt
From idea to living brand.
```

It should feel like:

```txt
empty space → spark → connection → digital mesh → identity → website reveal
```

It is the first emotional signal that the site is alive.

---

## 5. Hero Role

The Hero must communicate the core value instantly.

Main message:

```txt
We turn ideas into living digital brands.
```

The Hero should include:

- strong headline
- clear subheadline
- CTA to contact
- CTA to explore transformation
- animated digital mesh
- institutional tagline

---

## 6. Journey Role

The Journey explains the transformation.

It should not be a services list.  
It should show the evolution of a brand:

```txt
Spark
Structure
Identity
Digital Presence
Intelligence
Growth
```

---

## 7. Services Role

The Services section turns the narrative into a commercial offer.

It answers:

```txt
What does Lyken actually do?
```

Services:

- Brand Strategy
- Visual Identity
- Digital Presence
- Growth Marketing
- Web Platforms
- AI Solutions

---

## 8. Transformation Layers Role

This section shows the method behind the transformation.

Layers:

```txt
Clarity
Strategy
Identity
Technology
Growth
```

It communicates that Lyken works through structured layers, not random execution.

---

## 9. Process Role

The Process section builds trust.

Steps:

```txt
Discover
Define
Design
Build
Launch
Evolve
```

The user should feel that Lyken has a clear delivery method.

---

## 10. Solutions Role

Solutions by business stage help different clients identify themselves.

Stages:

```txt
New Brands
Repositioning
Digital Expansion
Intelligent Operations
```

---

## 11. Why Lyken Role

This section reinforces differentiation.

Main message:

```txt
Because modern brands need more than design.
```

It should explain why Lyken is not just design, marketing or technology in isolation.

---

## 12. Contact Role

The contact section is the conversion point.

It should make it easy to:

- start a project
- send a message
- select a service
- contact by WhatsApp if available

---

## 13. Navbar Behavior

Navbar must include:

```txt
Logo
About
Services
Process
Solutions
Contact
EN/PT
Start a Project
```

Rules:

- clear
- elegant
- not visually heavy
- sticky or fixed if implemented carefully
- mobile-friendly
- CTA always easy to find

---

## 14. Mobile Navigation

Mobile menu must include:

- main links
- language switcher
- CTA
- close button or clear close behavior

Rules:

- no clutter
- no dead-end
- no hidden language switcher
- links close menu after click

---

## 15. CTA Strategy

Primary CTA:

```txt
Start a Project / Iniciar um Projeto
```

Secondary CTA:

```txt
Explore the Transformation / Explorar a Transformação
```

CTA placement:

- Navbar
- Hero
- Journey or Services transition
- Final CTA
- Contact form

---

## 16. Bilingual UX

Language switcher:

```txt
EN / PT
```

Rules:

- always accessible
- visible on desktop
- available in mobile menu
- active state clear
- layout must not break in Portuguese

---

## 17. Scroll Experience

The scroll should feel like moving through a story.

Use:

- smooth reveal
- section rhythm
- visual continuity
- subtle transitions

Avoid:

- isolated blocks
- abrupt color jumps
- excessive parallax
- motion without meaning

---

## 18. Empty States

If any content is temporarily missing, use refined placeholders.

Do not show:

```txt
Lorem ipsum
coming soon
test
placeholder generic text
```

Use brand-safe temporary language if needed.

---

## 19. Conversion Goals

Primary conversion:

```txt
User sends a contact request.
```

Secondary conversions:

```txt
User understands services.
User explores transformation.
User trusts the brand.
User shares the website.
```

---

## 20. UX Audit Checklist

```txt
[ ] Visitor understands what Lyken does
[ ] Hero is clear
[ ] Journey tells transformation
[ ] Services are commercially clear
[ ] Process builds trust
[ ] Contact is easy
[ ] CTA is visible
[ ] EN/PT works
[ ] Mobile navigation works
[ ] No dead-end exists
[ ] Website feels alive, not static
```
