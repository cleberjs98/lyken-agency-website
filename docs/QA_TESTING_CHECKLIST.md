# Lyken Agency Website — QA Testing Checklist

## Purpose

This document defines the testing checklist for the Lyken Agency website before launch or phase approval.

Use this checklist to validate quality, consistency and readiness.

---

## 1. General Project QA

```txt
[ ] Project runs locally
[ ] No terminal errors
[ ] No critical console errors
[ ] Build completes successfully
[ ] Files are organized
[ ] Documentation is present
[ ] No unnecessary temporary files
[ ] No unused imports
[ ] No unnecessary console logs
```

---

## 2. Visual Identity QA

```txt
[ ] Deep Green is used as main background
[ ] Champagne Gold is used as accent
[ ] No random colors are used
[ ] Typography follows Playfair/Satoshi direction
[ ] Lines are thin and elegant
[ ] Cards look premium
[ ] Buttons look premium
[ ] Digital mesh is subtle
[ ] Texture does not hurt readability
[ ] Site does not look generic
```

---

## 3. Content QA

```txt
[ ] English copy is professional
[ ] Portuguese copy is natural and premium
[ ] No lorem ipsum
[ ] No placeholder copy in final view
[ ] CTAs are clear
[ ] Services are clear
[ ] Process is clear
[ ] Contact text is clear
[ ] Footer copy is correct
```

---

## 4. Bilingual QA

```txt
[ ] EN/PT switcher is visible
[ ] EN/PT switcher works
[ ] Active language is clear
[ ] Navbar changes language
[ ] Hero changes language
[ ] Services change language
[ ] Process changes language
[ ] Contact form changes language
[ ] Footer changes language
[ ] Portuguese does not break layout
```

---

## 5. Navigation QA

```txt
[ ] Navbar links work
[ ] CTA goes to contact
[ ] Secondary CTA works
[ ] Mobile menu opens
[ ] Mobile menu closes
[ ] Mobile links work
[ ] Scroll is smooth
[ ] Sticky navbar does not hide section titles
[ ] Footer links work
```

---

## 6. Loading QA

```txt
[ ] Loading appears
[ ] Loading is not generic spinner
[ ] Loading communicates idea/birth
[ ] LK monogram appears
[ ] Loading phrase is bilingual
[ ] Duration is not annoying
[ ] Transition to Hero is smooth
[ ] Loading does not freeze mobile
```

---

## 7. Hero QA

```txt
[ ] Headline is strong
[ ] Subheadline is readable
[ ] CTAs are visible
[ ] Mesh does not cover text
[ ] Hero looks premium
[ ] Hero works on mobile
[ ] Hero does not look like a template
```

---

## 8. Journey QA

```txt
[ ] Spark section exists
[ ] Structure section exists
[ ] Identity section exists
[ ] Digital Presence section exists
[ ] Intelligence section exists
[ ] Growth section exists
[ ] Journey feels sequential
[ ] Motion supports transformation
[ ] Text is not too long
```

---

## 9. Services QA

```txt
[ ] Six services exist
[ ] Service cards are clear
[ ] Icons are consistent
[ ] Cards are responsive
[ ] Text is concise
[ ] Visual style is premium
[ ] Services are commercially understandable
```

---

## 10. Contact QA

```txt
[ ] Contact section exists
[ ] Final CTA exists
[ ] Form fields exist
[ ] Labels are clear
[ ] Focus states exist
[ ] Validation messages exist or are planned
[ ] Mobile form works
[ ] No fake integration is presented as real
```

---

## 11. Responsiveness QA

Test widths:

```txt
[ ] 320px
[ ] 375px
[ ] 390px
[ ] 430px
[ ] 768px
[ ] 1024px
[ ] 1280px
[ ] 1440px
[ ] 1920px
```

Check:

```txt
[ ] No horizontal scroll
[ ] Text is readable
[ ] Buttons are tappable
[ ] Cards stack correctly
[ ] Menu works
[ ] Form works
[ ] Footer works
```

---

## 12. Accessibility QA

```txt
[ ] One H1 only
[ ] Headings follow order
[ ] Buttons are keyboard accessible
[ ] Links are keyboard accessible
[ ] Focus state is visible
[ ] Form labels exist
[ ] Contrast is acceptable
[ ] Decorative images do not confuse screen readers
[ ] Reduced motion considered
```

---

## 13. Performance QA

```txt
[ ] Images optimized
[ ] No oversized assets
[ ] Animations are not heavy
[ ] Mobile remains smooth
[ ] No unnecessary dependencies
[ ] Build output is acceptable
[ ] Loading does not delay excessively
```

---

## 14. SEO QA

```txt
[ ] Page title exists
[ ] Meta description exists
[ ] Open Graph title exists or is planned
[ ] Open Graph image exists or is planned
[ ] Favicon exists
[ ] Important images have alt text
[ ] Semantic HTML is used
```

---

## 15. Deploy QA

```txt
[ ] Build passes before deploy
[ ] Vercel deploy works
[ ] Public URL opens
[ ] Assets load online
[ ] Mobile online works
[ ] Console online has no critical errors
[ ] EN/PT works online
```

---

## 16. Final Approval Question

```txt
Does the website communicate birth, transformation, strategy, technology and growth through a premium living brand experience?
```
