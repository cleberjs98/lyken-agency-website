# 09 — Contact & Conversion

## Lyken Agency Website

## Objetivo desta fase

A Fase 09 tem como objetivo criar a área de contato e conversão do site.

Essa fase transforma interesse em ação.

A seção de contato deve ser clara, premium e fácil de usar.

---

# 09.1 — Função estratégica

O visitante deve chegar ao final da experiência pensando:

```txt
quero falar com a Lyken
quero iniciar um projeto
quero entender como isso se aplica ao meu negócio
```

A seção de contato deve facilitar essa ação sem fricção.

---

# 09.2 — CTA final

## English

```txt
Ready to transform your idea into a living brand?
```

## Português

```txt
Pronto para transformar sua ideia em uma marca viva?
```

## Texto de apoio EN

```txt
Let’s build the next stage of your business with strategy, design, technology and intelligence.
```

## Texto de apoio PT

```txt
Vamos construir a próxima etapa do seu negócio com estratégia, design, tecnologia e inteligência.
```

---

# 09.3 — Estrutura da seção

A seção deve conter:

- título forte
- texto curto
- formulário
- opção de contato direto
- detalhes institucionais
- elemento visual premium
- monograma LK discreto

---

# 09.4 — Formulário

Campos obrigatórios:

```txt
Name / Nome
Business Name / Nome da Empresa
Email / Email
Phone / Telefone
Service Needed / Serviço Desejado
Message / Mensagem
```

## Botão

EN:

```txt
Start a Project
```

PT:

```txt
Iniciar um Projeto
```

# 09.6 — Layout visual

A seção pode ser dividida em duas colunas no desktop:

```txt
coluna esquerda: CTA + texto + contato
coluna direita: formulário
```

No mobile:

```txt
CTA
texto
formulário
contato
```

---

# 09.7 — Visual do formulário

O formulário deve usar:

- fundo emerald green ou deep green
- bordas douradas finas
- inputs escuros
- labels claras
- foco dourado
- botão premium
- espaçamento confortável

## Evitar

- inputs brancos genéricos
- bordas cinza padrão
- formulário apertado
- excesso de campos
- visual de formulário comum

---

# 09.8 — Estados do formulário

Definir estados:

- default
- focus
- error
- success
- disabled
- loading submit

## Focus

O foco deve ser visível e elegante.

## Error

Erro deve ser claro, mas não agressivo.

## Success

Mensagem de sucesso deve ser elegante e objetiva.

---

# 09.9 — Validação

Validação mínima:

- nome não vazio
- email válido
- mensagem não vazia
- serviço selecionado, se obrigatório

Mensagens bilíngues.

---

# 09.10 — Integração inicial

Na primeira versão, o formulário pode ser apenas visual ou usar solução simples.

Opções futuras:

- EmailJS
- Formspree
- Resend
- API própria
- WhatsApp
- CRM
- Firebase/Supabase

## Regra

Se não houver integração real, deixar claro internamente na documentação.

---

# 09.11 — WhatsApp / contato direto

Pode haver botão secundário:

EN:

```txt
Contact us on WhatsApp
```

PT:

```txt
Falar pelo WhatsApp
```

## Regra

Não substituir o formulário, mas oferecer caminho rápido.

---

# 09.12 — Acessibilidade

Formulário deve ter:

- labels reais
- campos focáveis
- mensagens compreensíveis
- contraste suficiente
- ordem de tab correta
- botão claro

---

# 09.13 — Mobile

No mobile:

- campos devem ocupar largura total
- botões devem ser fáceis de tocar
- espaçamento deve ser confortável
- formulário não deve parecer comprimido
- CTA deve continuar forte

---

# 09.14 — O que a Fase 09 não deve fazer

Não precisa finalizar:

- CRM
- backend completo
- automações
- dashboard
- integrações avançadas

Foco:

```txt
conversão visual
clareza do contato
formulário pronto para integração
```

---

# 09.15 — Auditoria

Verificar:

- CTA final é forte?
- formulário está claro?
- campos são suficientes?
- visual é premium?
- mobile funciona?
- foco é acessível?
- mensagens são bilíngues?
- usuário sabe o que fazer?

---

# 09.16 — Critérios oficiais de aprovação

A Fase 09 só pode ser aprovada quando:

- CTA final existe
- formulário existe
- labels são bilíngues
- campos principais existem
- visual está alinhado ao branding
- mobile funciona
- estados de foco existem
- caminho de contato é claro
- não há confusão para o usuário

---

```txt
CTA final criado
Contact section criada
Formulário visual implementado
Campos bilíngues
Estados básicos definidos
Mobile validado
Caminho de conversão claro
```

# 09.18 — Pergunta de validação

```txt
A seção de contato transforma a experiência premium em uma ação clara e fácil para o visitante?
```

Se sim, aprovar.  
Se não, revisar CTA, formulário, clareza e mobile.
